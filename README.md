## Overview

This system combines a **rule-based heuristic engine** and an **XGBoost ML classifier**
into two hybrid strategies:

| Strategy  | Logic |
|-----------|-------|
| **Weighted** | `final_score = (0.8 × ML_prob) + (0.2 × rule_score)`; threshold ≥ 0.4 → phishing |
| **Fallback** | Trust ML when confident (≥ 0.6 or ≤ 0.4); defer to rules in ambiguous zone |

### Results (from project evaluation)

| Model | Accuracy | Precision | Recall | F1 |
|-------|----------|-----------|--------|----|
| Rule-Based Only | 0.6231 | 0.5220 | 0.1285 | 0.2063 |
| ML Only (XGBoost) | 0.9608 | 0.9472 | 0.9501 | 0.9487 |
| **Hybrid — Weighted** | **0.9749** | **0.9823** | **0.9672** | **0.9747** |
| Hybrid — Fallback | 0.9703 | 0.9907 | 0.9495 | 0.9697 |

## Setup

### 1 — Install dependencies

```bash
pip install -r requirements.txt
```

### 2 — Get the dataset

The original dataset is from Kaggle, cited from:

> Al-Subaiey, A., Al-Thani, M., Alam, N. A., Antora, K. F., Khandakar, A., & Zaman, S. A. U.
> (2024, May 19). *Novel Interpretable and Robust Web-based AI Platform for Phishing Email
> Detection*. arXiv. https://arxiv.org/abs/2405.11619

It is also available on Hugging Face:  
`https://huggingface.co/datasets/ealvaradob/phishing-dataset`

Place the raw CSV at `emails/raw_emails.csv`.  
Required columns: **`email_text`** (string) and **`label`** (0 = legit, 1 = phishing).

## Usage — Step by Step

### Step 1: Preprocess the raw dataset
```bash
python preprocess.py
```
Cleans HTML, URLs, special characters, lowercases text.  
Output → `emails/cleaned_ealvaradob_emails.csv`

---

### Step 2: Balance the dataset with SMOTE
```bash
python balance_with_smote.py
```
Applies SMOTE oversampling to equalise class distribution.  
Output → `emails/balanced_ealvaradob_emails.csv` + `models/vectorizer.pkl`

---

### Step 3: Train the XGBoost model
```bash
python train_model.py
```
Trains on TF-IDF features (unigrams + bigrams + trigrams, max 5000 features).  
Output → `models/phishing_xgboost_model.pkl` + updated `models/vectorizer.pkl`

---

### Step 4: Evaluate each component

```bash
# Rule-based engine
python evaluate_rule_based.py

# ML model only
python evaluate_ml_model.py

# Weighted hybrid (80% ML + 20% rules)
python evaluate_weighted_hybrid.py

# Fallback hybrid
python evaluate_fallback_hybrid.py
```

---

### Step 5: Predict on new emails

```bash
# ML only
python ml_predict.py

# Weighted hybrid (best performing)
python hybrid_weighted_predict.py

# Fallback hybrid
python hybrid_fallback_predict.py
```

Paste the email text at the prompt, then press **Ctrl+D** (Linux/macOS) or **Ctrl+Z** (Windows).

## Rule-Based Engine Details

The engine scores each email from 0–10 based on:

| Rule Category | Examples |
|---------------|---------|
| Suspicious keywords | "verify", "account locked", "action required" |
| Suspicious domains | `.xyz`, `.tk`, `bit.ly`, `tinyurl.com` |
| Brand spoofing | PayPal, Apple, Amazon, Microsoft, Google, Netflix |
| Suspicious link patterns | Shortened URLs, login/verify paths |
| Generic greetings | "Dear Customer", "Valued User" |
| Obfuscated addresses | `[at]`, `[dot]` |
| HTML tricks | `<form>`, `display:none`, `<script>`, `base64` |
| Urgency phrases | "final notice", "payment failed", "click to pay" |
| Sender spoofing | Fake support/admin addresses |

**Threshold:** score ≥ 2 → phishing
